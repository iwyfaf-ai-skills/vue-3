# Избегайте чрезмерной абстракции компонентов в больших списках

**Impact: СРЕДНЕЕ** - Экземпляры компонентов «тяжелее», чем простые DOM-узлы. Хотя абстракции
улучшают организацию кода, излишняя вложенность создает дополнительные накладные расходы. В больших
списках это многократно усиливается: 100 элементов с 3 уровнями абстракции означают 300+ экземпляров
компонентов вместо 100.

Не стоит полностью отказываться от абстракций, но важно помнить о глубине вложенности в часто
рендерящихся элементах, таких как элементы списков.

## Список задач

- Проверьте компоненты элементов списка на наличие ненужных компонентов-оберток
- Рассмотрите возможность упрощения иерархии компонентов в «горячих» путях
- Используйте нативные элементы, если компонент не добавляет реальной ценности
- Оцените количество компонентов с помощью Vue DevTools
- Сосредоточьте оптимизацию на наиболее часто рендерящихся компонентах

**ПЛОХО:**
```vue
<!-- ПЛОХО: Глубокая абстракция в элементах списка -->
<template>
  <div class="user-list">
    <!-- Для 100 пользователей: создается 400 экземпляров компонентов -->
    <UserCard v-for="user in users" :key="user.id" :user="user" />
  </div>
</template>

<!-- UserCard.vue -->
<template>
  <Card>  <!-- Компонент-обертка №1 -->
    <CardHeader>  <!-- Компонент-обертка №2 -->
      <UserAvatar :src="user.avatar" />  <!-- Компонент-обертка №3 -->
    </CardHeader>
    <CardBody>  <!-- Компонент-обертка №4 -->
      <Text>{{ user.name }}</Text>
    </CardBody>
  </Card>
</template>

<!-- Каждый UserCard создает: Card + CardHeader + CardBody + UserAvatar + Text
     100 пользователей = 500+ экземпляров компонентов -->
```

**ХОРОШО:**
```vue
<!-- ХОРОШО: Упрощенная структура элементов списка -->
<template>
  <div class="user-list">
    <!-- Для 100 пользователей: создается 100 экземпляров компонентов -->
    <UserCard v-for="user in users" :key="user.id" :user="user" />
  </div>
</template>

<!-- UserCard.vue — упрощенный, использует нативные элементы -->
<template>
  <div class="card">
    <div class="card-header">
      <img :src="user.avatar" :alt="user.name" class="avatar" />
    </div>
    <div class="card-body">
      <span class="user-name">{{ user.name }}</span>
    </div>
  </div>
</template>

<script setup>
defineProps({
  user: Object
})
</script>

<style scoped>
/* Стили, которые были бы в Card, CardHeader и т.д. */
.card { /* ... */ }
.card-header { /* ... */ }
.card-body { /* ... */ }
.avatar { /* ... */ }
</style>
```

## Когда абстракция все же оправдана

```vue
<!-- Абстракция компонента полезна, когда: -->

<!-- 1. Инкапсулируется сложное поведение -->
<UserStatusIndicator :user="user" />  <!-- Содержит логику, подсказки и т.д. -->

<!-- 2. Компонент переиспользуется вне «горячего» пути -->
<Card>  <!-- Хорошо для редких случаев, но не для списка из 100 элементов -->

<!-- 3. Сам список небольшой -->
<template v-if="items.length < 20">
  <FancyItem v-for="item in items" :key="item.id" />
</template>

<!-- 4. Используется виртуализация (одновременно рендерится ~20 элементов) -->
<RecycleScroller :items="items">
  <template #default="{ item }">
    <ComplexItem :item="item" />  <!-- Хорошо — существует только 20 экземпляров -->
  </template>
</RecycleScroller>
```

## Измерение накладных расходов на компоненты

```javascript
// В режиме разработки профилируйте количество компонентов
import { onMounted, getCurrentInstance } from 'vue'

onMounted(() => {
  const instance = getCurrentInstance()
  let count = 0

  function countComponents(vnode) {
    if (vnode.component) count++
    if (vnode.children) {
      vnode.children.forEach(child => {
        if (child.component || child.children) countComponents(child)
      })
    }
  }

  // Для точных подсчетов используйте Vue DevTools
  console.log('Проверьте вкладку Components в Vue DevTools для подсчета экземпляров')
})
```

## Альтернативы компонентам-оберткам

```vue
<!-- Вместо компонента <Button> для стилизации: -->
<button class="btn btn-primary">Нажми меня</button>

<!-- Вместо компонента <Text>: -->
<span class="text-body">{{ content }}</span>

<!-- Вместо компонентов-оберток для верстки внутри списков: -->
<div class="flex items-center gap-2">
  <!-- контент -->
</div>

<!-- Используйте CSS-классы или Tailwind вместо компонентов-оберток для стилизации -->
```

## Расчет влияния на производительность

| Размер списка   | Компонентов на элемент | Всего экземпляров  | Влияние на память |
|-----------------|------------------------|--------------------|-------------------|
| 100 элементов   | 1 (плоская структура)  | 100                | Базовый уровень   |
| 100 элементов   | 3 (вложенная)          | 300                | ~3x по памяти     |
| 100 элементов   | 5 (глубоко вложенная)  | 500                | ~5x по памяти     |
| 1000 элементов  | 1 (плоская структура)  | 1000               | Высокое           |
| 1000 элементов  | 5 (глубоко вложенная)  | 5000               | Очень высокое     |
