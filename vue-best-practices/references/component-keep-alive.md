# Практические рекомендации по использованию компонента KeepAlive

**Impact: ВЫСОКИЙ** - `<KeepAlive>` кэширует экземпляры компонентов вместо их уничтожения.
Используйте его для сохранения состояния при переключениях, но явно управляйте размером и
актуальностью кэша, чтобы избежать утечек памяти или устаревшего интерфейса.

## Task List

- Используйте KeepAlive только там, где сохранение состояния улучшает UX
- Установите разумное значение `max`, чтобы ограничить размер кэша
- Объявляйте имена компонентов для сопоставления include/exclude
- Используйте `onActivated` / `onDeactivated` для логики, учитывающей кэширование
- Решите, как и когда кэшированные представления обновляют свои данные
- Избегайте кэширования компонентов, потребляющих много памяти, или связанных с безопасностью

## Когда использовать KeepAlive

Используйте KeepAlive при переключении между представлениями, где состояние должно сохраняться
(вкладки, многошаговые формы, панели управления). Избегайте его, когда каждый визит должен
начинаться с чистого состояния.

**ПЛОХО:**
```vue
<template>
  <!-- Состояние сбрасывается при каждом переключении -->
  <component :is="currentTab" />
</template>
```

**ХОРОШО:**
```vue
<template>
  <!-- Сохраняется состояние при переключениях -->
  <KeepAlive>
    <component :is="currentTab" />
  </KeepAlive>
</template>
```

## Когда НЕ использовать KeepAlive

- Страницы поиска или фильтрации, где пользователи ожидают свежих результатов
- Компоненты, потребляющие много памяти (карты, большие таблицы, медиаплееры)
- Чувствительные сценарии, где данные должны быть очищены при выходе
- Компоненты с интенсивной фоновой активностью, которую вы не можете приостановить

## Ограничьте и контролируйте кэш

Всегда ограничивайте размер кэша с помощью `max` и по возможности сужайте кэширование до конкретных
компонентов.

```vue
<template>
  <KeepAlive :max="5" include="Dashboard,Settings">
    <component :is="currentView" />
  </KeepAlive>
</template>
```

## Убедитесь, что имена компонентов соответствуют include/exclude

`include` и `exclude` сопоставляются с опцией name компонента. Явно задавайте имена для надёжного
кэширования.

```vue
<!-- TabA.vue -->
<script setup>
defineOptions({ name: 'TabA' })
</script>
```

```vue
<template>
  <KeepAlive include="TabA,TabB">
    <component :is="currentTab" />
  </KeepAlive>
</template>
```

## Стратегии инвалидации кэша

В Vue 3 нет прямого API для удаления конкретного кэшированного экземпляра. Используйте ключи или
динамические include/exclude, чтобы принудительно обновлять.

```vue
<script setup>
import { ref, reactive } from 'vue'

const currentView = ref('Dashboard')
const viewKeys = reactive({ Dashboard: 0, Settings: 0 })

function invalidateCache(view) {
  viewKeys[view]++
}
</script>

<template>
  <KeepAlive>
    <component :is="currentView" :key="`${currentView}-${viewKeys[currentView]}`" />
  </KeepAlive>
</template>
```

## Хуки жизненного цикла для кэшированных компонентов

Кэшированные компоненты не уничтожаются при переключении. Используйте хуки активации для обновления
и очистки.

```vue
<script setup>
import { onActivated, onDeactivated } from 'vue'

onActivated(() => {
  refreshData()
})

onDeactivated(() => {
  pauseTimers()
})
</script>
```

## Кэширование роутера и актуальность

Решите, должна ли навигация показывать кэшированное состояние или свежее представление.
Распространённый паттерн — использовать ключ по маршруту, когда параметры меняются.

```vue
<template>
  <router-view v-slot="{ Component, route }">
    <KeepAlive>
      <component :is="Component" :key="route.fullPath" />
    </KeepAlive>
  </router-view>
</template>
```

Если вы хотите повторно использовать кэш, но получать свежие данные, обновляйте их в `onActivated`
и сравнивайте query/params перед запросом.
